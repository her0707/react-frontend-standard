# Coding Patterns Template

Use this template as the starting point for `docs/coding-patterns.md`.

## Responsibility Table Reminder

Always include a simple table covering:

- `screens`
- feature `components`
- feature `hooks`
- feature `api`
- feature `service`
- feature `schema`
- feature `type`
- shared `components`

## Baseline Rule

- `screen` shows
- `component` renders
- `hook` connects
- `api` calls
- `service` orchestrates
